Design spec · v2 (pivoted)

Talk to me — a voice AI of Noah

2026-05-20 · branch talk-to-me · scope: play-around, not production-polish
Pivot from v1: dropped the DIY Pipecat + Fly.io pipeline in favor of ElevenLabs Conversational AI (their managed voice-agent product). Reason: it collapses the architecture to a single embed + a single Netlify Function. No second service to operate, no Python, no separate deploy. Trade-off is more vendor lock-in and less pipeline control, both acceptable for play-around scope.

Goal

Add a Talk to me button to the homepage. Clicking it opens a real-time voice conversation with an AI version of Noah — input via microphone, output in a cloned version of Noah's voice. Every conversation is transcribed and stored so Noah can review what people asked.

This is a personal experiment, not a polished product. The bar is "interesting to play with," not "ready for thousands of users."

Decisions baked in

Static site stays on Netlify with no build changes. The agent's UI is a web component that opens its own WebSocket directly to ElevenLabs from the browser. Our only backend involvement is receiving the post-call webhook and a private page to read the logs.

Components

1. Homepage button + embed claude builds

  • Add a button to src/pages/index.astro styled in the existing aesthetic (no generic CTA chrome). Hover state uses the oxblood accent.
  • Add a small disclosure line near the button: "Heads up — your voice is recorded by ElevenLabs and the transcript is saved to my server so I can review it." Plain English about what's happening; matters for trust and for California two-party consent hygiene (even though the user knows they're talking to an AI).
  • Clicking the button reveals the ElevenLabs widget inline below it (not a modal). One way to do this: 
    <elevenlabs-convai
  agent-id={import.meta.env.PUBLIC_ELEVENLABS_AGENT_ID}
  variant="full"
></elevenlabs-convai>
<script src="https://unpkg.com/@elevenlabs/convai-widget-embed"
        async type="text/javascript"></script>
    The agent ID is a public identifier and ships in the page HTML. This is fine only if the agent has an allowed-origins list configured (so someone can't paste our agent ID into their own site and burn our monthly minute cap). If ElevenLabs doesn't expose an allowed-origins setting at the agent level, the monthly cap is the only thing protecting the bill — to be verified during impl, see risks below.
  • The widget's default chrome ships with ElevenLabs branding and styling. We override what we can via the documented attributes/CSS variables to look closer to the site's aesthetic. If the widget styling proves too constrained, fall back to building a custom shell using their React/JS SDK (still no new service — just more frontend code).

2. The agent itself (configured in ElevenLabs dashboard) you configure

One-time setup in the ElevenLabs web UI:

  • Voice: the voice ID of your Instant Voice Clone.
  • LLM: Anthropic Claude (their dropdown). You can either use ElevenLabs' bundled LLM key (simpler; rolled into their per-minute price) or supply your own Anthropic key (more granular billing). Recommend bundled for v0.
  • System prompt: first-person persona doc — written by you, with optional first-draft help from me using your blog posts.
  • Knowledge base: upload the markdown bodies of your blog posts (Markdown + legacy posts) plus a short bio file. ElevenLabs runs its own RAG over these on each turn.
  • Limits: set max session duration (e.g., 5 min) and a monthly minute cap. Both configurable in their dashboard.
  • Post-call webhook: point at https://noahlandesberg.com/api/log-conversation with a shared secret header.

3. Log-ingest Netlify Function claude builds

  • New file: netlify/functions/log-conversation.ts.
  • Verifies the shared-secret header (env var ELEVENLABS_WEBHOOK_SECRET). Drops anything else with 401.
  • Parses the ElevenLabs post-call payload. Their webhook delivers the full transcript, metadata, and duration when a conversation ends.
  • Writes a normalized JSON blob to Netlify Blobs at conversations/<YYYY-MM-DD>/<conversation_id>.json: 
    {
  "conversation_id": "elabs_abc123",
  "started_at": "2026-05-20T18:32:04Z",
  "duration_s": 142,
  "transcript": [
    { "role": "user", "ts": 0.0, "text": "what do you do?" },
    { "role": "noah", "ts": 1.4, "text": "I work on..." }
  ],
  "ended_by": "user" | "time_cap" | "error",
  "raw": { /* full original payload, in case we want fields later */ }
}
  • Returns 204. No business logic. If the write fails we 500 and let ElevenLabs retry per their webhook policy.

4. Admin review page claude builds

  • New Astro page: /talk-to-me-logs. Not linked from anywhere.
  • Gated by a shared-secret query param (?key=…) matched against an env var. Identity / OAuth is overkill for a private personal page.
  • Page reads from Netlify Blobs via a small Netlify Function (list-conversations.ts) that returns the index + transcripts on demand.
  • Newest-first list with date, duration, first user line as preview. Click → full transcript view styled like a chat log.

5. Cost guardrails

All in the ElevenLabs dashboard, not in our code:

  • Max session duration (per call). Their pipeline ends the call when hit.
  • Monthly minute cap on the agent. Their pipeline refuses new sessions when hit.
  • Per-IP / abuse limits are not first-class in their dashboard at this writing — to be verified during impl. If absent and it matters, we add a thin Netlify Function in front of the widget mount that mints short-lived signed agent tokens and rate-limits by IP.

Sequence of one conversation

1. User clicks "Talk to me"
   → widget mounts, opens WSS to ElevenLabs with the public agent-id
2. ElevenLabs streams a greeting in your cloned voice
3. Loop:
   - user speaks → ElevenLabs STT
   - transcript → Claude (with your knowledge base in context)
   - response → cloned-voice TTS streamed back
4. User clicks End OR session timer expires
5. ElevenLabs POSTs the full conversation payload to
   /api/log-conversation with shared-secret header
6. Function writes the JSON blob; review at /talk-to-me-logs?key=…

What you (Noah) need to do

  1. you Create an ElevenLabs account if you don't have one. Pick a tier that includes Conversational AI minutes (Starter usually does, but verify current pricing).
  2. you Record a ~1-minute clean voice sample. Upload to ElevenLabs as an Instant Voice Clone. Save the voice ID.
  3. you In the ElevenLabs dashboard, create an Agent: select Claude as the LLM, your cloned voice, paste the system prompt, attach the knowledge base files. (I'll prep the prompt + KB files in the impl phase; you just paste/upload.)
  4. you Set the agent's per-session and monthly limits.
  5. you Configure the post-call webhook URL + shared secret in ElevenLabs.
  6. you If ElevenLabs offers an allowed-origins (domain allowlist) setting on the agent, set it to noahlandesberg.com and the Netlify deploy-preview pattern. This stops anyone from re-embedding your agent on their own site. If they don't, the monthly minute cap is the backstop.
  7. you Add Netlify env vars: PUBLIC_ELEVENLABS_AGENT_ID, ELEVENLABS_WEBHOOK_SECRET, TALK_TO_ME_LOGS_KEY.
  8. you Approve the persona doc I draft before it's pasted into ElevenLabs.

That's it. No Fly.io, no Anthropic key on your side (ElevenLabs covers it via the bundled LLM tier — unless you want to bring your own).

Out of scope for v0

  • Mobile-optimized UI. Desktop browser first; widget should work on mobile but layout polish is later.
  • Cross-session memory. Each conversation starts fresh.
  • Custom moderation pass. We rely on Claude's default refusal behavior plus the persona doc to handle scope.
  • Multi-language. English only.
  • Replay of audio in the admin page. Transcripts only — ElevenLabs may or may not include audio URLs in the webhook payload; we won't store it either way.
  • Custom-built widget UI. v0 uses their stock widget; we restyle if their CSS hooks allow it.

Known risks & open questions

  • Agent ID embed abuse. The agent ID is in the page's HTML. Without an allowed-origins list on the agent, anyone can paste the tag into their own site and consume our minutes. Mitigation order: (1) allowed-origins setting if ElevenLabs supports it, (2) tight monthly cap, (3) if abuse appears, switch to a token-mint pattern where a Netlify Function issues short-lived signed tokens checked against IP/referrer before the widget connects.
  • Audio storage on ElevenLabs side. Even though we only persist transcripts, ElevenLabs likely retains the raw audio on their servers (per their default retention). Verify their current retention policy during impl and reflect it in the homepage disclosure copy so visitors aren't surprised.
  • Widget styling limits. The embedded widget may not theme into the site's aesthetic cleanly. Plan B is to build our own shell with their JS SDK (still no new backend). v0 is "stock widget, lightly themed."
  • Per-IP rate limiting. If their dashboard doesn't expose this, we add a tiny Netlify Function that issues short-lived signed tokens to mount the widget. To be verified during impl, not blocking the spec.
  • Voice quality. Instant Voice Clone is OK but not amazing. If the result is uncanny in a bad way, upgrade to Professional Voice Clone (~30 min sample).
  • Persona drift. Claude will happily make up biographical details if asked. The system prompt must include "if you don't know a specific fact, say so" and the knowledge base must include factual anchors (resume, project list, recent posts).
  • Webhook reliability. If ElevenLabs fails to deliver the post-call webhook, that conversation isn't logged. Mitigation for v0: rely on their retries; if we see gaps we can also poll their conversations API as a backup.
  • Vendor lock-in. The whole conversation lives inside ElevenLabs. Migration cost later = nontrivial. Accepted for play-around scope.
  • Public disclosure of recording. The disclosure copy next to the button must make recording obvious, both for trust and for legal hygiene.

Success criteria for v0

  • Click button → speak → hear a reply in your cloned voice end-to-end, with the latency ElevenLabs delivers out of the box (typically ~1s).
  • End the session → full transcript appears in /talk-to-me-logs?key=… within a few seconds.
  • Hit the 5-minute cap → session ends gracefully via ElevenLabs.
  • Run for a week without bill surprise — usage stays within the monthly minute cap.
